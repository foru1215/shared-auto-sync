# 24か月キャリア戦略計画書

**対象期間**: 2026/3/14 〜 2028/3/31
**最終更新**: 2026/3/14
**管理方法**: GitHub Projects + Codex 自動同期

---

## 1. 全体方針（BLUF）

### 最重要戦略

**「制御盤営業 → 制御・設備 × AI の実務人材」への転換を 24か月で完遂する。**

資格（第一種電気工事士 → 電験三種）で基礎信用を獲得し、制御・PLC の実務理解で技術者と対等に会話できる力を積み、Codex / Claude Code の実践スキルで「AI を設備・制御に適用できる人材」として差別化する。

### なぜこの順番か

1. **資格が先** — 学歴不足を最短で補う手段は国家資格。第一種電気工事士（2026/4-7）で「電気の基本を理解している」証明を取り、電験三種で「理論レベルで理解している」信用を上乗せする
2. **制御基礎が次** — 現職で日常的に触れる制御盤の図面・仕様を「読めて語れる」状態にすることで、技術者ポジションへの接続性が生まれる
3. **AI が差別化** — 資格と制御理解は他にもいる。Codex / Claude Code を制御・設備領域で使いこなせる人材はまだ少ない。ここが最大の差別化要素

### 狙う職種

| 優先度 | 職種 | 理由 |
|--------|------|------|
| 本命 | 生産技術（制御・電気寄り） | 年収の伸び大、役職ルート明確、制御盤営業からの接続性高い |
| 対抗 | 設備開発・設備設計 | 技術深度が評価されやすい、AI活用との相性良好 |
| 安全本命 | 電気設計・制御設計 | 資格＋制御理解で応募可能、実務経験を積みやすい |

### 学歴不足の埋め方

1. **第一種電気工事士** — 実技まで取得し「手を動かせる」証明
2. **電験三種** — 理論的裏付けの証明（在学中でも「科目合格」で評価される）
3. **制御基礎ノート・図面読解ノート** — 自走学習の証拠
4. **現職での技術寄り実務証拠** — 営業でも技術側に踏み込んだ実績
5. **AI活用ポートフォリオ** — 他候補者にない武器

### AI の差別化戦略

- 「AI を触れます」ではなく「AI で制御図面の整理を自動化しました」「仕様書の比較チェックを Claude Code で構築しました」と言えるレベルを目指す
- GitHub 上に学習ログ・成果物を蓄積し、面接で URL を渡せる状態にする
- Codex / Claude Code のスペシャリストとして、設備・制御領域での AI 適用事例を持つ

## 2. 転職戦略の中核

### 本命：生産技術（制御・電気寄り）

**狙い方**: 製造業の生産技術部門。制御盤メーカー営業で培った「仕様会話力」「客先折衝力」に加え、制御理解・資格・AI活用を武器にする。

- **年収の伸び**: 500万〜800万（係長〜課長クラスで700万超も可能）
- **役職ルート**: 主任 → 係長 → 課長。生産技術は現場寄りで成果が見えやすく、昇進しやすい
- **現職経験の活かし方**: 顧客折衝・仕様会話の経験は、社内の設計部門・外注先とのやり取りにそのまま転用できる
- **制御理解不足の埋め方**: PLC 基礎 + シーケンス制御 + 図面読解で、生産設備の制御を理解する

### 対抗：設備開発・設備設計

**狙い方**: 自動車・半導体・食品等の製造業で、設備の企画〜導入を担当する部門。

- **年収の伸び**: 500万〜900万（専門性が評価されると高い）
- **役職ルート**: リーダー → マネージャー。技術深度＋マネジメント力の両方が求められる
- **現職経験の活かし方**: 制御盤の仕様理解、ベンダーとの折衝経験が直結
- **制御理解不足の埋め方**: 設備全体のシステム構成を理解する。制御盤は設備の一部であることを意識

### 安全本命：電気設計・制御設計

**狙い方**: 制御盤メーカー、エンジニアリング会社、プラントエンジニアリング会社。

- **年収の伸び**: 450万〜700万
- **役職ルート**: 技術リーダー → 設計課長。資格が直接評価される
- **現職経験の活かし方**: 制御盤の仕様・図面に日常的に触れている経験が活きる
- **制御理解不足の埋め方**: 図面を「読む」から「描く」へステップアップ

## 3. 24か月マスタープラン

### Phase 0: 準備・直前対策期（2026/3/14 〜 2026/4/27）

**目的**: 第一種電気工事士学科試験の合格と、24か月計画の基盤構築

**やること**:
- 第一種電気工事士の学科対策（過去問周回、苦手分野の集中学習）
- GitHub Projects の初期セットアップ（Epic / Milestone / Label 体系の構築）
- Codex / Claude Code の基本操作習得（環境構築、基本コマンド）
- 学習環境の整備（ノートPC のセットアップ、ツール導入）

**やらないこと**:
- 電験三種の学習
- PLC の深い学習
- 転職活動

**この時期に優先する理由**: 4/27 の学科試験が直近の最重要マイルストーン。ここで落とすと全体計画が 1 年ずれる。

**月次・週次への接続**: 週次で過去問の正答率を記録し、70% 未満の分野を翌週の重点にする。

**面接で効く証拠**: 「計画的に資格取得に取り組んだ」「GitHub で学習管理をしていた」と語れる。

### Phase 1: 実技集中期（2026/4/28 〜 2026/7/4）

**目的**: 第一種電気工事士の実技試験合格

**やること**:
- 実技試験の候補問題練習（全13問）
- 工具の扱い、ケーブル加工、圧着、接続の反復練習
- 制御基礎の入門学習（実技と並行できる軽い学習）
- Codex / Claude Code の学習継続（1日15分程度）

**やらないこと**:
- 電験三種の本格学習
- PLC の深い学習
- 転職活動

**この時期に優先する理由**: 実技試験の合否が資格取得を決定する。落とすと翌年まで持ち越し。

**月次・週次への接続**: 週2回以上の実技練習を確保。候補問題を1週間に2問ペースで完成させる。

**面接で効く証拠**: 「学科・実技ともに一発合格」は計画力と実行力の証明。

---

### Phase 2: 制御基礎強化期（2026/7/5 〜 2026/10/31）

**目的**: 制御の基礎を固め、現職の業務をより深く理解できるようにする

**やること**:
- 制御基礎の体系学習（リレーシーケンス、自己保持回路、インターロック等）
- 制御図面読解の基礎（展開接続図、単線結線図の読み方）
- 発電機盤・ポンプ盤の仕様理解深化
- 制御基礎ノートの作成開始
- Claude Code Academy の学習開始
- 現職での技術寄り実務証拠づくり開始

**やらないこと**:
- 電験三種の本格学習（様子見で教材確認程度）
- 転職活動

**この時期に優先する理由**: 資格取得後、次に必要なのは「制御を理解している」という実質。ここが弱いと、どの職種にも応募できない。

**月次・週次への接続**: 月1テーマで制御基礎を学び、各テーマの理解をノートにまとめる。週末に図面読解練習。

**面接で効く証拠**: 制御基礎ノート、図面読解ノートが「自走で学べる人材」の証拠になる。

---

### Phase 3: PLC / 図面読解強化期（2026/11/1 〜 2027/3/31）

**目的**: PLC とシーケンス制御を実践レベルで理解し、図面を「読める」から「語れる」へ

**やること**:
- PLC 基礎学習（ラダー図の読み書き、基本命令）
- シーケンス制御の体系学習
- 図面読解の深化（制御盤の展開接続図を詳細に読む）
- PLC シミュレータでの課題作成
- 電験三種の段階導入（理論科目から開始）
- AI 活用の実務適用開始（仕様書整理、技術メモ作成の効率化）

**やらないこと**:
- 転職活動の本格化
- 電験三種の全科目同時学習

**この時期に優先する理由**: 制御基礎の上に PLC / シーケンスを積む順序が自然。この時期に電験三種も段階導入し、理論の基盤を固める。

**月次・週次への接続**: PLC 課題を月2つ以上作成。電験三種は週末に理論科目の学習。

**面接で効く証拠**: PLC 課題のリポジトリ、シーケンス制御の理解を示すノート。

---

### Phase 4: 電験三種段階導入期（2027/4/1 〜 2027/9/30）

**目的**: 電験三種の科目合格を狙いつつ、制御・AI スキルを深化

**やること**:
- 電験三種の学習本格化（理論 → 電力 → 機械 → 法規の順序）
- 2027年度の電験三種試験に向けた計画的学習
- PLC 応用課題（タイマー、カウンタ、データ処理）
- AI 活用の中級スキル習得（Claude Code での自動化構築）
- 実務証拠の蓄積継続

**やらないこと**:
- 全科目同時に完璧を目指す（科目合格狙い）
- 転職活動の本格化

**この時期に優先する理由**: 電験三種は「取得済み」でなくても「科目合格中」で評価される。この時期に着手すれば、2027年度試験で2〜3科目合格が狙える。

**月次・週次への接続**: 電験三種は平日30分＋週末2時間。PLC・AI は週2回。

**面接で効く証拠**: 電験三種の科目合格通知、学習ログ。

---

### Phase 5: AI スペシャリスト強化期（2027/10/1 〜 2028/1/31）

**目的**: AI 活用を差別化レベルに引き上げ、転職準備の仕上げに入る

**やること**:
- Codex / Claude Code の高度な活用（ワークフロー自動化、コード生成、分析）
- 制御・設備領域での AI 適用事例の構築

- 電験三種の残り科目学習継続

**やらないこと**:
- AI 学習の趣味化（必ず制御・設備と接続する）

**この時期に優先する理由**: 資格・制御基礎・PLC が揃った上で AI を武器化することで、「資格だけの人」「AI だけの人」との差別化が成立する。

**月次・週次への接続**: 月1つ以上の AI 活用成果物を作成。転職準備は週末に。

**面接で効く証拠**: GitHub 上の AI 活用リポジトリ、制御 × AI の適用事例。

**目的**: 転職活動の本格化と内定獲得

**月次・週次への接続**: 週2社以上の応募。面接準備は日次で。

## 5. 週次運用ルール

### 平日の標準運用（月〜金）

| 時間帯 | 内容 | 目安時間 |
|--------|------|----------|
| 通勤時間 | 参考書・テキストの読み込み | 30分 |
| 昼休み | 過去問1〜2問 or 用語確認 | 15分 |
| 帰宅後 | 集中学習（その月の最重要テーマ） | 45〜60分 |
| **合計** | | **約1.5〜2時間/日** |

### 週末の標準運用（土・日）

| 時間帯 | 内容 | 目安時間 |
|--------|------|----------|
| 午前 | メイン学習（資格 or 制御 or PLC） | 2時間 |
| 午後 | サブ学習（AI or 実務証拠 or 図面読解） | 1.5時間 |
| 夕方 | 週次レビュー（日曜のみ） | 30分 |
| **合計** | | **約3.5〜4時間/日（週末合計 7〜8時間）** |

### 週次合計目安

- 平日: 7.5〜10時間
- 週末: 7〜8時間
- **週合計: 約15〜18時間**

### 毎週のレビュー方法（日曜夕方 30分）

1. GitHub Projects の Issue を確認し、完了タスクをクローズ
2. 今週の学習時間と内容を振り返る
3. 来週の最重要タスクを3つ決める
4. 遅れがあれば、翌週の計画を調整
5. 月末成果物の進捗を確認

### 学習が遅れたときの立て直し方

1. **1週間の遅れ**: 翌週のサブテーマを削り、メインテーマに集中
2. **2週間の遅れ**: 月の目標を縮小し、最重要タスクだけに絞る
3. **1か月の遅れ**: フェーズ計画を見直し、次フェーズの開始を1か月ずらす
4. **致命的な遅れ**: 転職仕上げ期を延長（2028/Q2 までバッファ）

### 試験前と試験後の優先度切替ルール

- **試験2か月前**: その資格に学習時間の 80% を配分
- **試験1か月前**: その資格に学習時間の 90% を配分
- **試験翌週**: 1週間はリフレッシュ週。軽い学習のみ
- **試験翌月**: 次のフェーズの最重要テーマに切り替え

## 8. 実務証拠の設計

営業職の立場から作れる実務証拠を、難易度順に整理する。

### 難易度 1（すぐ着手可能）

#### 技術メモの作成

- **内容**: 日常業務で触れる技術情報を自分の言葉でメモに残す
- **期待効果**: 技術への関心と理解力を示す
- **面接での使い方**: 「日常的に技術情報を整理する習慣があります」
- **今の私で着手可能か**: はい。今日からできる

#### 仕様書の技術視点での読み直し

- **内容**: 既存案件の仕様書を技術視点で精読し、制御方式・回路構成をメモ
- **期待効果**: 仕様書を「読める」ことの証明
- **面接での使い方**: 「仕様書を技術的に理解し、設計者と対等に会話できます」
- **今の私で着手可能か**: はい。既存の仕様書にアクセスできる

### 難易度 2（少し踏み込む）

#### 現場同行レポート

- **内容**: 制御盤の据付・試運転への同行時に、技術的な観察をレポート化
- **期待効果**: 現場を見ている実績、技術的な観察力
- **面接での使い方**: 「現場で制御盤の動作確認に立ち会い、記録を残していました」
- **今の私で着手可能か**: はい。同行の機会があれば

#### 客先との技術会話の記録

- **内容**: 客先との打ち合わせで技術的な内容を記録し、社内にフィードバック
- **期待効果**: 技術会話に参加できる力の証明
- **面接での使い方**: 「客先の技術要件をヒアリングし、社内の設計部門に共有していました」
- **今の私で着手可能か**: はい。日常業務の延長

### 難易度 3（意識的に作る）

#### 図面確認の実績

- **内容**: 納品前の図面確認に参加し、チェック結果を記録
- **期待効果**: 図面を読める能力の証明
- **面接での使い方**: 「制御盤の図面チェックにも参加し、誤記や仕様不整合を指摘した経験があります」
- **今の私で着手可能か**: 上司に相談すれば機会を得られる可能性あり

#### 発電機盤・ポンプ盤の比較整理資料

- **内容**: 2種類の制御盤の仕様・制御方式の違いを比較表にまとめる
- **期待効果**: 制御盤の種類ごとの特性を理解していることの証明
- **面接での使い方**: 「担当製品の技術的な違いを自ら整理し、理解を深めました」
- **今の私で着手可能か**: はい。仕様書を基に作成可能

### 難易度 4（中長期で取り組む）

#### 社内改善提案

- **内容**: 業務プロセスの改善提案を技術的な視点から作成
- **期待効果**: 問題発見力と改善力の証明
- **面接での使い方**: 「営業の立場から技術プロセスの改善を提案しました」
- **今の私で着手可能か**: テーマを見つけてから。半年以内に1件

#### AI を使った補助資料・業務改善

- **内容**: Claude Code / Codex を使って仕様書の比較、技術メモの整理等を効率化
- **期待効果**: AI を実務に適用できる能力の証明
- **面接での使い方**: 「AI ツールを使って業務を改善した実績があります」
- **今の私で着手可能か**: AI の基本操作習得後（2026/7 以降）

## 10. リスクと対策

### リスク 1: 第一種電気工事士で予定がずれる

| 項目 | 内容 |
|------|------|
| 発生原因 | 学科または実技の不合格 |
| 兆候 | 過去問正答率が試験2週間前で60%未満 |
| 回避策 | 過去問を5年分×3周以上回す。正答率を毎週記録し、70%未満の分野を翌週に集中 |
| 修正策 | 不合格の場合、翌年の再受験を計画に組み込む。他の学習は予定通り進める |

### リスク 2: 電験三種を広げすぎる

| 項目 | 内容 |
|------|------|
| 発生原因 | 4科目を同時に進めて全て中途半端になる |
| 兆候 | どの科目も過去問正答率が40%未満 |
| 回避策 | 科目合格狙い。最初は理論+電力の2科目に集中 |
| 修正策 | 科目を1つに絞り、確実に1科目合格する |

### リスク 3: AI 学習が趣味化する

| 項目 | 内容 |
|------|------|
| 発生原因 | AI の学習が楽しくなり、制御・資格の学習時間を侵食 |
| 兆候 | 資格・制御の学習時間が計画の50%を下回る |
| 回避策 | AI 学習は必ず制御・電気と接続する。「AI 単体」の学習は制限する |
| 修正策 | AI 学習を週2回に制限し、資格・制御に時間を戻す |

### リスク 4: 制御基礎が浅いまま進む

| 項目 | 内容 |
|------|------|
| 発生原因 | ノートを作っただけで理解が伴わない |
| 兆候 | 図面を見ても回路の動作を説明できない |
| 回避策 | 月末に「口頭で説明できるか」のセルフテストを実施 |
| 修正策 | 次のフェーズに進む前に、制御基礎の理解度を確認。不十分なら1か月延長 |

### リスク 5: 実務証拠が作れない

| 項目 | 内容 |
|------|------|
| 発生原因 | 現職で技術寄りの業務に関われない |
| 兆候 | 3か月経っても実務証拠が1つもない |
| 回避策 | 難易度1（技術メモ、仕様書精読）から始める。上司への相談不要で着手可能 |
| 修正策 | 実務証拠のハードルを下げ、「仕様書を読んで整理した」レベルから積み上げる |

### リスク 6: GitHub Projects が形骸化する

| 項目 | 内容 |
|------|------|
| 発生原因 | Issue の更新が面倒になり、放置される |
| 兆候 | 2週間以上 Issue の更新がない |
| 回避策 | 週次レビュー（日曜30分）を習慣化。Codex で自動化できる部分は自動化 |
| 修正策 | Issue の粒度を大きくして更新負荷を下げる。月次レビューに切り替えも可 |

### リスク 7: 転職準備が後ろ倒しになる

| 項目 | 内容 |
|------|------|
| 発生原因 | 学習に集中しすぎて、転職準備を先送りにする |
| 兆候 | 2027/12 時点で職務経歴書のドラフトがない |
| 回避策 | 2027/11 を転職準備開始の Hard Deadline に設定 |
| 修正策 | 学習の一部を削り、転職準備に振り替える。計画は2028/Q2まで延長可能 |
