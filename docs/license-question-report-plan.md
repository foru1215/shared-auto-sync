# License Question And Report Plan

## 対象

- 第一種電気工事士 学科
- 期間: `2026-03-15` 〜 `2026-04-27`
- 試験日: `2026-04-27`

## この期間の考え方

この期間は、`量を回す` と `弱点を見える化する` の 2 つだけに絞ります。

やることは毎回同じです。

1. 問題を解く
2. 点数を記録する
3. 間違いの理由を書く
4. 次に解き直す範囲を決める

## 問題数の基本ルール

- 平日: `20問`
- 土曜: `30問`
- 日曜: `30問` + 週次レビュー

これで 1 週間の標準は `160問` です。

## 報告先

- 日次の学習報告: `ISS-001` と `ISS-005`
- 週次のまとめ: `ISS-R01`

運用ルール:

- `2026-03-31` までは `ISS-001`
- `2026-04-01` 以降は `ISS-005`
- 毎週日曜に `ISS-R01` へ週次まとめ

## 日次報告テンプレート

この形で Issue コメントに残します。

```md
## 2026-03-15

- 解いた問題数: 20問
- 正解数: 12問
- 正答率: 60%
- 間違い分類:
  - 知識不足: 5
  - ひっかけ: 2
  - 計算 / 手順ミス: 1
- 今日の弱点 Top3:
  - 例: 電気理論
  - 例: 法規
  - 例: 用語の取り違え
- 次回やること:
  - 例: 今日間違えた 8 問を解き直す
  - 例: 法規を 10 問追加で解く
```

## 週次報告テンプレート

これは `ISS-R01` に残します。

```md
## Week 1 | 2026-03-15 〜 2026-03-21

- 目標: 120問
- 実績: 110問
- 累計: 110問 / 960問
- 週平均正答率: 62%
- よく落とした分野 Top3:
  - 例: 法規
  - 例: 配線設計
  - 例: 計算問題
- 未達理由:
  - 例: 平日に 2 日止まった
- 来週の修正:
  - 例: 平日は毎日 20 問を死守
  - 例: 土曜に苦手分野を 30 問まとめてやる
- 次週の最優先:
  - 例: 法規
  - 例: 誤答の解き直し
  - 例: 正答率 70% まで引き上げ
```

## 間違い分類ルール

毎回この 3 つで分けます。

- 知識不足
- ひっかけ / 読み違い
- 計算 / 手順ミス

これを毎回記録すると、ただの点数ではなく「何を直せばいいか」が見えます。

## 進捗の見方

順調かどうかは、次の 3 つで見ます。

- 問題数が週目標に近い
- 正答率が上がる
- 弱点が毎週減る

## 最低ライン

忙しい日でも、これだけは守るラインです。

- `10問` は解く
- 間違い理由を 1 行で残す
- 次回やる問題を 1 つ決める

## 一言でいうと

`毎日 20 問前後を解いて、点数よりも「何を間違えたか」を報告する`
