## SSS 判定理由

1. **Issue 単位で何をするかが明確** — タスクリストが seed の `work_steps` そのまま。ボイラープレートなし。
2. **完了条件が観測可能** — 「完了している」ではなく「正答率が数値で記録されている」レベル。
3. **Win Condition が子 Issue と連動** — 抽象的な「関連カードの順番を決める」ではなく ISS-xxx の完了確認。
4. **最初の一手が文法的に正しい** — 47 箇所の「ために」崩れを修正。
5. **completion_check からボイラープレート除去** — 47 箇所の汎用行を削除。
6. **全 85 Issue が収束** — apply 後の re-dry-run で全件 noop。
7. **テストが通る** — validate / pytest / audit すべてパス。
